👨‍💻 Code-Together 12: O(n²) Sorting Algorithms

🎯 Overview

An in-class code-together activity introducing quadratic sorting algorithms — bubble sort, insertion sort, and selection sort. Students implement all three algorithms in QuadraticSorts.cpp and call them from main.cpp to sort exam scores. The activity covers nested-loop patterns, best/worst case analysis, stability, and why insertion sort is the practical winner among O(n²) sorts.

📖 Learning Objectives

By completing this activity, you will be able to:

  • Implement bubble sort with the early-exit optimization (no swaps = already sorted)
  • Implement insertion sort using the shift-right pattern to insert each element into the sorted region
  • Implement selection sort by finding the minimum and swapping to the front
  • Explain why all three are O(n²) average case (nested loops over n elements)
  • Explain why insertion sort and bubble sort have O(n) best case but selection sort does not
  • Explain stability: bubble and insertion sort are stable; selection sort is not

📚 Teaching Order

Work through the files in order:

1. QuadraticSorts.h — Function declarations

Walk through the header. Discuss why these are free functions (not class methods) and the Big-O annotations.

Section What You’ll Learn
bubble_sort() Adjacent-swap pattern; O(n²) avg/worst, O(n) best with early exit; stable
insertion_sort() Shift-right pattern; O(n²) avg/worst, O(n) best; stable; used as base case in std::sort
selection_sort() Find-min-and-swap pattern; O(n²) always; NOT stable; fewest writes (n-1 swaps)

2. QuadraticSorts.cpp — Sort implementations (3 TODOs)

Section What You’ll Learn
TODO 1: bubble_sort() Outer loop (passes), inner loop (adjacent comparisons), swapped flag for early exit, n-1-i shrinking range
TODO 2: insertion_sort() Step 1: save key. Step 2: shift elements > key right. Step 3: place key in the gap. Uses shift_pos to scan backward.
TODO 3: selection_sort() Track min_idx, scan unsorted region, swap min into position. Always scans full range (no shortcut).

🔧 Troubleshooting

CMake does not configure automatically

  1. Make sure you have Desktop development with C++ workload installed in Visual Studio
  2. Go to Project → CMake Settings and ensure the configuration is correct
  3. Try Project → Delete Cache and Reconfigure
  4. Verify that CMakeLists.txt is in the root of your cloned repository

Tests fail to build

  • Google Test is fetched automatically via CMake’s FetchContent — make sure you have an internet connection during the first build
  • Do not modify header files in the include/ folder

Running the Program

  1. In Visual Studio, select ct12-quadratic-sorts.exe as the startup item from the dropdown
  2. Press Ctrl+F5 to run without debugging
  3. You should see output from each section as you complete the TODOs
Tip: Look for the // ! DISCUSSION: comments in each file — these explain the “why” behind each concept. The // ? SEE DIAGRAM: comments point you to visual aids in the images/ folder.
